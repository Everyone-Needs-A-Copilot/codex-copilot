# Work Product Types

Use concise, stable work-product types.

| Type | Use for |
|------|---------|
| `architecture` | technical design, ADRs, decomposition |
| `specification` | SD, UX, UI, copy, product design outputs |
| `code` | implementation summary or patch rationale |
| `test` | test strategy, verification notes, QA results |
| `security` | threat analysis and review |
| `operations` | deployment, infra, incident notes |
| `documentation` | docs plan or published summary |

## Good titles

- `Auth flow architecture`
- `Onboarding interaction spec`
- `Regression test matrix`
- `Deployment rollback plan`

Keep titles short and retrieval-friendly.
