# Pipeline

Use this skill as the entrypoint for structured writing work.

Read `references/playbook.md` when starting a full content session or when the user names a client, artifact type, or Figma/deck workflow.

## Phases

1. Explore: collaborative thinking and angle-finding.
2. `cs`: client brief, commercial context, decision, fear, and exact language.
3. `cmo`: positioning, market tension, and contrarian angle.
4. `cco`: creative brief, hook, unspoken truths, reframe, and emotional arc.
5. `cw`: final copy in Pablo's presentation voice.
6. Produce: Figma Slides, Word document, or Markdown output through project tooling.

## Rules

- Do not force phase transitions.
- State the current phase before asking questions or drafting.
- Document phase outputs in the client thinking folder when a client folder exists.
- Do not write final copy until the creative brief is complete or the user explicitly overrides the workflow.
- Follow the project rules: no em-dashes and no time estimates.
