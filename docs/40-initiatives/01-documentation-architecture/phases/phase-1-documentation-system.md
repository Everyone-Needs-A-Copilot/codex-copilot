# Phase 1: Documentation System

## Objective

Replace the flat documentation collection with a numbered, navigable system and make initiative documentation a standard project capability.

## Work Definition

1. Reclassify existing pages as tutorial, how-to, explanation, reference, troubleshooting, or appendix.
2. Move canonical content into numbered sections and archive a map of former public paths.
3. Consolidate duplicate pack, release, setup, and usage surfaces.
4. Add `docs/40-initiatives/` indexes and templates to the framework and generated projects.
5. Update shared documentation standards and project instructions.
6. Resolve target-project access to the QA gate.
7. Verify links, generated output, tests, and QA-gate behavior.

## Exit Criteria

- numbered documentation sections and indexes resolve
- former public documentation paths are mapped to canonical pages under `docs/90-archive/redirects/`
- setup creates initiative scaffolding and a working QA-gate link without overwriting user files
- shared standards define `docs/40-initiatives/` and the `tc` ownership boundary
- parity and smoke tests pass
- TASK-22 has an evidence-bound passing QA work product
