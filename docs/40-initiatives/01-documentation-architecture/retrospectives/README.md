# Retrospective

## What Worked

- Convoco provided a proven initiative structure instead of requiring a speculative format.
- Consolidating former paths into one archive map kept migration history discoverable without leaving redirect clutter at the documentation root.
- A real temporary-project fixture caught setup behavior more reliably than prose inspection alone.
- Separating initiative knowledge from `tc` execution state resolved the apparent conflict with The Markdown Memory.

## What Changed

- Documentation architecture became a tested framework contract rather than a style preference.
- Project setup now provides initiative and QA-gate infrastructure on first install.
- The shared documentation standard now applies the convention to future projects.

## Carry Forward

- Every formal initiative should define its validation contract before implementation begins.
- Existing project initiative directories must be preserved exactly during framework updates.
- Documentation claims that describe executable behavior should be tested through generated output, not only link checks.
